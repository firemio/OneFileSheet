# One File Sheet

One File Sheet は、1つのHTMLファイルだけで動く簡易スプレッドシートです。

サーバーやインストールは不要です。表データは `OneFileSheet.html` 内の JSON として保存され、ブラウザの `localStorage` は使いません。

![One File Sheet (Terminal テーマ)](screenshot.png)

## デモ

**[▶ ブラウザでデモを開く](https://firemio.github.io/OneFileSheet/OneFileSheet.html)**（GitHub Pages）

デモはそのまま編集・貼り付け・アンドゥなどを試せます。編集した内容は、ダウンロードボタンの「HTML をエクスポート」でアプリ＋全データ入りのHTMLファイルとして保存できます。以後はそのファイルを Chrome / Edge で開けば、「上書き保存」でファイル自身に直接保存できます。

## なぜ作ったか

ちょっとした表データが xls や Google スプレッドシートに入っていると、AI エージェントに中身を探させたり、編集内容の diff を取ったりするのが面倒です。

One File Sheet は、現代の LLM がそのまま扱いやすい HTML に、表計算アプリとデータを1ファイルでまとめます。アプリ本体とデータが同じファイルにあるため、AI エージェントで読み書きしやすく、応用や共有もしやすくなります。

## 特徴

- 1ファイル完結のHTMLアプリ
- サーバー不要
- `localStorage` 不使用
- **複数シート対応**(タブで切り替え、追加・名前変更・並べ替え・削除)
- **タイトル編集**(画面上のタイトルをクリックして編集、ファイルに保存)
- **テーマ32種 + 自動**(右上のボタンから選択。Light / Dark / Sepia / Nord / Dracula / Solarized など。選択はファイルに保存され、共有先でも同じ見た目)
- **UI言語: 英語 / 日本語**(既定はブラウザの言語設定に追従。右上の言語ボタンで切替、選択はファイルに保存)
- ファイルパスの表示
- セルの直接編集(プレーンテキストのみ、書式は混入しない)
- 行番号・列レター表示、行/列の挿入・削除(行番号・列名クリックでメニュー)
- **クイック統計**(各行・各列の両端にあるグラフアイコンにホバーで合計・平均・件数・最大・最小を表示。クリックで固定表示になり、編集に追従してライブ更新。セルには何も書き込まない読み取り専用ビュー)
- キーボード操作: Enter/Tab/矢印キーでセル移動、Alt+Enter でセル内改行、最終行で Enter を押すと行を自動追加
- Excel / Google スプレッドシートからの範囲貼り付け(改行入りセルの引用符形式にも対応、行・列は自動拡張)
- 元に戻す / やり直し(Ctrl+Z / Ctrl+Y)
- Ctrl+S で上書き保存、未保存インジケータ(ステータス表示とタブタイトルの ●)、未保存のままタブを閉じようとすると警告
- 「再読み込み」ボタンでファイルの最新内容を取り込み(AI エージェントがファイル側を編集した後などに)
- 保存時にファイル側のデータが外部で変更されていれば警告(AI エージェントとの同時編集を検知)
- **HTMLエクスポート**(アプリ＋全データ入りの自分自身の複製をダウンロード。File System Access API 非対応のブラウザやデモページでも保存手段になる)
- CSV書き出し(シートごと、BOM付きUTF-8、Excelで文字化けしない)
- 選択したHTMLファイルへの上書き保存
- Chrome / Edge 対応(閲覧・編集・HTMLエクスポートは Firefox / Safari でも可)

## 使い方

1. `OneFileSheet.html` を Chrome または Edge で開きます。
2. 表を編集します。
3. 「上書き保存」または Ctrl+S を押します。初回はファイル選択ダイアログが開くので、`OneFileSheet.html` 自身を選ぶと、そのまま保存されます。
4. 2回目以降は「上書き保存」だけで同じHTMLファイル内のデータが更新されます。

別の OneFileSheet ファイルを編集したい場合は、そのファイル自体をブラウザで開くか、開いているページにファイルをドラッグ&ドロップします(アプリはファイルに同梱されています)。

保存先はブラウザの IndexedDB に記録されるため、次回以降は保存先が復元されます(記録はファイルのURLごとに分かれるため、コピーした複数のシート間で保存先が混ざることはありません)。権限が残っていれば、そのまま「上書き保存」できます。ブラウザが再許可を求めた場合は、画面の許可操作に従ってください。

## データの保存場所

表データは、HTML内の次の要素に JSON として保存されます。

```html
<script id="sheet-data" type="application/json">
...
</script>
```

そのため、編集後の `OneFileSheet.html` 自体がデータファイルでもあります。

データ形式は次のような JSON オブジェクトです(v2)。

```json
{
  "title": "One File Sheet",
  "theme": "auto",
  "activeSheet": 0,
  "sheets": [
    { "name": "Sheet1", "data": [["見出し1", "見出し2"], ["値", "値"]] }
  ]
}
```

各シートの `data` は「文字列の2次元配列」で、先頭行が見出し行です。旧形式(2次元配列のみ)のファイルも読み込み時に自動で移行されます。保存時にはセル内の `<` が `\u003c`(JSONのユニコードエスケープ)にエスケープされるため、セルに HTML タグ(`</script>` を含む)や `$` 記号などを入力してもファイルが壊れることはありません。

AI エージェント向けの編集規約は、HTML内の sheet-data ブロック直前にある `AGENT NOTES` コメントに記載しています。

## 注意

このツールは File System Access API を使って、ユーザーが選択したHTMLファイルへ書き込みます。

Firefox / Safari では File System Access API に対応していない場合があるため、Chrome または Edge での利用を推奨します。

ブラウザの仕様上、HTMLファイル自身を自動で勝手に書き換えることはできません。初回の保存時にファイル選択ダイアログで保存対象のファイルを選択し、保存許可を与える必要があります。

保存先のファイルハンドルは IndexedDB に記録します。ただし、ブラウザの状態や権限の期限によっては、再度許可が必要になることがあります。
